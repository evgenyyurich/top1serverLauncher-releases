# Авто-новости из Discord → `content.json`

Новости и девблог лаунчера постятся сами: пишешь в Discord — через несколько минут появляется
в лаунчере. Ручного редактирования JSON не нужно. Скрипт **сам определяет тип канала** (см. «Два режима»).

```
 Discord форум-канал        GitHub Action (крон ~10 мин)        content.json         лаунчер
 ┌───────────────┐  бот       ┌──────────────────────────┐      ┌──────────┐  HTTPS  ┌────────┐
 │ #news / #dev  │  читает    │ scripts/build-content.mjs │ ───▶ │ commit   │ ──────▶ │ Новости│
 │ тред = запись │ ─────────▶ │ треды → JSON             │      │ в репо   │  (старт)│ Девблог│
 └───────────────┘            └──────────────────────────┘      └──────────┘         └────────┘
```

Механику делают два файла в этом репозитории:
`.github/workflows/sync-content.yml` (когда запускать) и `scripts/build-content.mjs` (что собирать).
**Правки лаунчера не требуются** — он уже читает `content.json` (v1.2.0+).

## Два режима (определяются автоматически)
Скрипт смотрит тип канала и берёт записи по-разному:

- **Форум-канал** (тип «Форум») → **тред = карточка**: название треда → заголовок, первый пост → текст,
  тег треда → категория (`Обновление`/`Ивент`/`Техработы` → **цветной** бейдж) или сборка девблога.
  Аккуратнее всего — рекомендуется для новых каналов.
- **Канал-объявлений / текстовый** → **сообщение = карточка**: первая строка → заголовок, остальное → текст.
  Тегов/категорий нет (бейдж нейтральный «Новость»). Удобно, если новости уже постятся в обычный канал.

> Текущая настройка top1server: `📡▪️новости` (объявления) + `dev-blog-ru` (текстовый) — работают во
> **втором режиме**. Форум-шаги ниже нужны, только если решишь перейти на форум-каналы.

## Настроить один раз

### 1. Форум-каналы
В Discord создай два **форум-канала** (Создать канал → тип «Форум»):
- один для новостей (напр. `новости`),
- один для девблога (напр. `девблог`) — опционально.

Тред в форуме = одна запись: **название треда → заголовок**, **первый пост → текст-тизер**.

**Теги форума** (Настройки канала → Теги) задают бейдж:
- Новости: заведи теги `Обновление`, `Ивент`, `Техработы` (можно по-английски `update`/`event`/`maintenance`).
  Именно они дают **цветной** бейдж; любой другой тег покажется серым как есть.
- Девблог: тег = номер сборки (напр. `1.21.11`) — он рисуется пилюлей на карточке.

Ставь **один** тег на тред (берётся первый).

### 2. Бот (только чтение)
1. https://discord.com/developers/applications → **New Application**.
2. Вкладка **Bot** → **Reset Token** → скопируй токен (понадобится в шаге 4).
3. Там же **Privileged Gateway Intents** → включи **MESSAGE CONTENT INTENT**.
   Без него заголовки/даты/теги работают, но **тизеры (текст) будут пустыми**.
4. Вкладка **OAuth2 → URL Generator**: scope `bot`, права **View Channels** + **Read Message History**.
   Открой полученную ссылку и добавь бота на сервер. Проверь, что бот видит оба форум-канала.

> Рекомендуется **отдельный бот только на чтение**, а не бот регистрации из прокси: у Action-а
> будет минимум прав, и токен не даёт ничего, кроме чтения этих каналов (CLAUDE.md §4 — это не
> клиентский секрет). Переиспользовать бота прокси можно, но его токен куда чувствительнее.

### 3. ID сервера и каналов
В Discord: Настройки → Расширенные → включи **Режим разработчика**. Затем ПКМ по:
- серверу → **Копировать ID сервера** → это `GUILD_ID`;
- форум-каналу новостей → **Копировать ID канала** → `NEWS_CHANNEL_ID`;
- форум-каналу девблога → `DEVBLOG_CHANNEL_ID` (если делаешь девблог).

### 4. Secrets и Variables в GitHub
Этот репозиторий → **Settings → Secrets and variables → Actions**:

| Тип | Имя | Значение |
|-----|-----|----------|
| **Secret** | `DISCORD_TOKEN` | токен бота из шага 2 |
| Variable | `GUILD_ID` | ID сервера — **опц.**, вычисляется из `NEWS_CHANNEL_ID`, можно не задавать |
| Variable | `NEWS_CHANNEL_ID` | ID форум-канала новостей |
| Variable | `DEVBLOG_CHANNEL_ID` | ID форум-канала девблога (опц.) |
| Variable | `NEWS_LIMIT` | сколько новостей держать (по умолч. 12) |
| Variable | `DEVBLOG_LIMIT` | сколько записей девблога (по умолч. 8) |

Токен — только в **Secret** (шифруется, не виден в логах). ID — не секрет, кладём в Variables.

### 5. Проверка
Вкладка **Actions** → workflow **«Sync content from Discord»** → **Run workflow**.
Через полминуты глянь `content.json` — новости из тредов должны появиться. Дальше он
запускается сам каждые ~10 минут.

## Как пользоваться дальше
Просто пиши тред в форум-канале, ставь тег — и всё. Отредактировал/удалил тред — при следующей
синхронизации подхватится. Латентность до лаунчера: ~10 мин (крон) + следующий старт лаунчера у игрока.

## Что берётся из треда
| поле JSON | откуда в Discord |
|-----------|------------------|
| `title` | название треда |
| `summary` | первый пост, plain-text, обрезка до ~200 симв. |
| `category` (news) / `build` (devblog) | первый тег треда |
| `date` | время создания треда |
| `url` | ссылка на тред («читать полностью») |
| `id` | ID треда |

## Баннер и магазин
`banner` и `shop` в Discord не живут — Action их **не трогает**, только `news` и `devblog`.
Правь `banner`/`shop` прямо в `content.json` (коммитом), они сохранятся при каждой синхронизации.

## Безопасность
- Workflow запускается только по крону и вручную — **не** на `pull_request`, чтобы форк-PR не мог
  получить токен. Не добавляй `pull_request`-триггер.
- Бот — только чтение (View Channels + Read Message History) и только на нужных каналах.
- Токен нигде не логируется; скрипт печатает только счётчики.

## Если что-то не так
- **Тизеры пустые, заголовки есть** → не включён MESSAGE CONTENT INTENT (шаг 2.3).
- **Run падает `Discord 401`** → неверный/протухший `DISCORD_TOKEN`.
- **`Discord 403` / пусто** → бот не видит канал (права/не добавлен на сервер).
- **Ничего не запускается** → не заданы `DISCORD_TOKEN`/`GUILD_ID`/`NEWS_CHANNEL_ID`; до этого
  Action намеренно завершается зелёным no-op (в логе «not configured — skipping»).
- Логи: вкладка **Actions** → последний запуск → шаг «Build content.json from Discord».
